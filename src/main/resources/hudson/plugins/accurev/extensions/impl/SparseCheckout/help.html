Some projects only use a portion of the stream or reference tree and, in these instances, it is not necessary to check out the entire contents of the repository
In such cases, it may be useful to only populate what's specified in the sparse checkout.

Directories and files must be specified one per line and must use relative pathnames. For example, you could use:

src/com/my-company/main/my-file.java
src/com/test
src/com/webapp/my-file2.xml

where "src" is the immediate sub-folder under your stream or reference tree.

NOTES:

  • If you specify a spare checkout, with Poll SCM selected, where you do not specify any Inclusion or Exclusion files or directories,
    then the Sparse checkout files and directories are used as the filter for polling.
    In this case, changes made to elements in the Sparse checkout will trigger a build in Jenkins.
